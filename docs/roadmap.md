# 로드맵 — MVP 이후

MVP 범위 밖의 확장 사항. **확정(방향 정해짐)** 과 **OPEN(아직 미결)** 을 구분한다. MVP 범위·TODO는 [mvp-plan.md](mvp-plan.md) / [mvp-todo.md](mvp-todo.md).

---

## ✅ 확정된 확장 사항 (post-MVP, 방향 결정됨)

구현은 MVP 이후지만 설계 방향은 결정 원장에 확정된 것들.

### 에이전트/기능 확장

- **CS 응대 서브그래프** (결정 1) — 대화형 문의 응대. MVP 구매자 그래프에 분기로 합류 예정.
- **리뷰 분석 에이전트** (결정 10/10-A) — 이벤트/배치 기반 독립 그래프. 리뷰 1건당 3산출:
  - (a) 악성 리뷰 판정·분류 (판정까지만, 조치는 Spring/운영자)
  - (b) 상품/판매자 리뷰 인사이트 집계 (측면별 감성) → 추천·판매자 답변 데이터 소스
  - (c) 작성자 취향 신호 → 프로필 write 후보 (승격 게이트 경유)
- **상품 질문 분기 / 일반 대화 조건부 추천** (결정 17/18) — 리뷰 집계 조회, 부정 시 추천 재호출, 일반 대화 high/low 조건부 추천.
- **판매자 고도화** — 리뷰 인사이트 기반 답변, 상세페이지 편집 보조 확장 (MVP는 통계 Q&A + 단순 draft).

### 프로필

- **Temporal Knowledge Graph (프로필 v2)** — 완전한 시간 인식 지식 그래프. MVP는 OKF 자연어 위키 + recency-wins로 유예.

### 검색/인프라

- **하이브리드 검색** — pgvector만으로 키워드 정확도가 부족할 경우 Elasticsearch/BM25 하이브리드 도입 (재검토 조건: 카탈로그 규모 확대 또는 정확도 요구 상승).
- **AI 자체 벡터 검색 강화** — 질의 시점 Spring 위임에서 AI 임베딩 비중 확대 (후보 흐름 OPEN과 연동).
- **다중 인스턴스 확장** — 동시 스트림 레지스트리·레이트 리밋 카운터를 in-memory → Redis 이관 (MVP는 단일 인스턴스 전제).

### UX/운영

- **가격 홀드 UX** (결정 9-B) — 추천 표시 이후 가격 변경(TOCTOU) 시 표시가 N분 보장 vs 결제 전 재검증. 기획·Spring 협의.
- **FE 채팅 히스토리 복원** — 새로고침/재접속 시 이전 대화 렌더. 지원 시 AI가 `GET /chat/history?sessionId=` 제공 (대화 원본은 AI 소유).

## ❓ OPEN — 아직 정하지 않은 결정들

방향이 확정되지 않아 실측·협의·정책 판단이 필요한 항목.

### 설계 미결

| # | 항목 | 무엇을 정해야 하나 | 어떻게 정할까 |
|---|---|---|---|
| O-1 | **질의 시점 후보 흐름** | 방식1(AI 벡터 검색 → Spring id 제약 조회) vs 방식2(Spring 검색 → 임베딩 재랭킹 보조) | `SearchBackend`로 양쪽 구현 후 골든셋/NDCG@10 실측 |
| O-2 | **verifiedSum vs 표시가** | 검색 응답가 기준 예산 합산과 Spring enrich 표시가의 순간 괴리 UX 정책 | 기획·Spring 협의 (질의 시점이라 괴리 창 축소됨) |
| O-3 | **채팅 히스토리 복원** | 새로고침 시 대화 유실 수용 여부 → API 신설 필요 여부 | FE 팀 확인 |
| O-4 | **규약 문자열 포맷** | 조건 칩 제거 문자열(`"[조건 제거] priceMax"`) 정확 포맷 | LLM 팀(우리) 확정 후 FE 통보 — 즉시 가능 |
| O-5 | **헬스/운영 엔드포인트** | `/health` 외 readiness/metrics 노출 범위 | 배포 구성 시 결정 |

### Spring 협의 잔여 (계약 미확정)

MVP 착수를 막는 것은 [mvp-todo.md](mvp-todo.md#-spring-협의-대기-블로킹) 참조. 아래는 계약 세부 잔여.

- **C-3**: I-2에 재고 코드(`OUT_OF_STOCK`) 부재 — 재고 검증 여부·코드 추가. `CART_OPTION_REQUIRED` options 목록 스키마(optionId + 표시명).
- **C-5**: `productId` 문자열 통일 — BE/FE 예시가 숫자라 양팀 재통보 필요.
- **C-11**: CORS 허용 오리진 목록 (레이트 리밋은 확정).
- **C-1**: `role` 값 집합(member/guest/seller 문자열)·토큰 TTL/갱신 정책.
- **C-13/C-14**: I-6 `metric`/`groupBy` 값 집합, I-7 편집 가능 필드·소유권 검증.

### 폐기되어 다시 열지 않는 것 (참고)

- 카탈로그 이벤트/웹훅(CDC) 동기화 → I-8 pull 배치로 영구 대체.
- 주문 알림/미러(`/events/order`) → `GET /orders/recent` 질의 시점 조회로 영구 대체.
- 상품 원본 컬럼의 AI측 사본(미러) → 미채택. AI는 생성물만 저장.
- 비회원 3회 제한 → 폐기(무제한 검색/추천, 장바구니는 게스트 허용·구매만 회원).
